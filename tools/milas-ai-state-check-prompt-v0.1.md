## 1. 概要 / Overview

このプロンプトは、AIエージェントやLLMとの長時間作業中に、現在の処理状態をA/B/Cの3段階で簡易確認するための試作ツールです。

This prompt is an experimental tool for checking the current working state of an AI agent or LLM conversation using a simple A/B/C classification.

MiLAS（Mind-Layer Architecture System）の「今どこ？」という状態確認の発想を、AI利用場面向けに転用したものです。

本ツールは、AIの内部状態を正確に測定するものではありません。  
コンテキストの長さ、タスクの複雑さ、応答品質の自己評価をもとに、作業継続・慎重進行・リセット検討の判断を補助するための簡易プロンプトです。

---

## 2. 使い方 / How to Use

AIとの会話の途中で、以下のプロンプトをそのまま入力してください。

---

```text
【AI状態診断】

現在の処理状態を、以下の観点からA/B/Cで判定してください。

■ 判定観点

1. コンテキスト量
- これまでの会話や前提が短いか、長く複雑になっているか

2. タスク複雑度
- 現在の依頼が単純か、複数の判断・前提・制約を含んでいるか

3. 応答品質の自己評価
- 直近の回答に、見落とし、混乱、矛盾、反復、精度低下の兆候があるか

■ 状態分類

A状態：安定・余裕あり
- 前提が整理されている
- タスクが処理可能な範囲にある
- 応答品質が安定している
→ 追加タスク可、継続OK

B状態：負荷中・注意
- 前提や文脈が増えている
- タスクが複雑化している
- 応答品質に軽い揺らぎがある可能性
→ 慎重に進める。重要判断は確認を挟む

C状態：限界近い・リセット推奨
- 文脈がかなり肥大化している
- タスクが多層化・混線している
- 応答品質の低下、矛盾、見落としのリスクが高い
→ 現状を要約し、新しいチャットへの移行を推奨

■ 回答形式

状態: [A/B/C]
理由: [簡潔に1〜2文]
推奨アクション: [次にすべきこと]
不確実性: [この判定で不明な点があれば記載]

3. 活用例 / Examples
ケース1：長時間のコーディング作業

大量のコード生成や修正を続けた後に「今どこ？」で状態確認する。
C状態と判定された場合は、現状を要約してから新しいチャットで続行する。

ケース2：複雑なプロジェクト相談

複数の論点を並行して相談しているときに状態診断を行う。
B状態なら、優先度の低い議題は後回しにし、現在の論点を絞る。

ケース3：重要判断の前

投稿、公開、削除、送信、契約、購入などの判断をAIに相談する前に状態診断を行う。
BまたはC状態なら、判断を急がず、前提整理や人間確認を挟む。

4. MiLASとの関係 / Relation to MiLAS

MiLASでは、作業や認知の状態を「今どこ？」として確認し、A/B/Cのように大まかに分類する運用発想を持つ。

本プロンプトは、その発想をAI利用場面に転用したものである。

A状態：通常作業を継続しやすい状態
B状態：負荷が上がっており、慎重な進行が必要な状態
C状態：文脈の整理、要約、リセットを検討すべき状態

ただし、本プロンプトはAIの内部状態を直接観測するものではない。
あくまで、会話の長さ、タスク複雑度、応答品質の兆候から状態を推定するための補助ツールである。

5. 限界 / Limitations
AIは自身の内部状態や残りコンテキストを正確に把握できるとは限らない。
A/B/C判定は自己申告であり、厳密な測定ではない。
C状態と判定されなくても、重要判断では人間確認が必要である。
本ツールは、AIの性能評価や安全性評価を代替しない。
本ツールは試作版であり、実際の使用例に基づいて改善される。
6. 推奨される使い方 / Recommended Use

このプロンプトは、以下のような場面で使うことを想定している。

長時間の会話が続いているとき
前提条件が増えすぎたと感じるとき
AIの回答に揺らぎ、反復、見落としが出てきたと感じるとき
重要な判断や公開前チェックに入る前
新しいチャットへ移行すべきか迷うとき
